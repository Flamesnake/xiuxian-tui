# 终端修仙模拟器 — MVP 实现计划（v2）

## Context

在终端 TUI 中实现一款"仪表盘修仙模拟器"休闲游戏。用户提供了深度设计反馈，核心诉求：
- **资源拮抗**：属性之间要有 trade-off，让每次操作都有权衡
- **时间压力**：寿元是真正的稀缺资源，给玩家紧迫感
- **叙事沉浸**：事件文案有色彩、有因果链、有仙侠味
- **突破仪式感**：境界突破不是简单数值变化，要有视觉反馈和策略选择
- **操作爽感**：快捷键、批量操作、即时反馈

技术栈：Python 3.12.7 + **Textual 8.2.7**（仪表盘式多面板 UI 的最佳选择）

---

## 一、核心玩法循环

### 属性体系（拮抗关系设计）

| 属性 | 作用 | 获得方式 | 消耗方式 |
|---|---|---|---|
| **修为** | 突破境界的核心进度 | 修炼、丹药、奇遇 | 突破失败反噬 |
| **气血 (HP)** | 容错率，过低影响修炼效率 | 休息、丹药 | 探索受伤、走火入魔 |
| **灵气** | 行动资源，特殊操作消耗 | 冥想、聚灵丹、自然恢复 | 修炼、炼丹、炼器 |
| **寿元** | 硬性时间限制（年） | 突破境界大幅增加 | 每次行动消耗 |
| **灵石** | 货币 | 探索、坊市交易、任务 | 购买丹药/材料 |

**核心拮抗链**：
- 苦修（安全但耗时多 → 扣寿元多） vs 顿悟（耗时少但耗灵气多 + 概率走火入魔）
- 探索（高回报随机事件）→ 可能受伤 → 需要休息恢复（休息又耗寿元）
- 寿元紧张 → 想快速突破 → 嗑药 → 消耗灵石 → 需要更多探索/交易

### 时间系统

- 每次操作消耗**可变时间**：休息=0.01年(~3.65天)，修炼=0.1年(~36.5天)，闭关=0.25年(~3个月)，探索=0.05年(~18天)
- 初期寿元 **40-60年**（而非80年），小突破+1年，大境界突破+10~30年
- 玩家始终感知"不突破就会老死"的紧迫感

### 数值曲线

- 修为需求：`基础值 × (层数^1.5)`，前期快后期慢，迫使玩家探索多途径
- 事件强度随境界动态调整（妖兽伤害、奇遇收益）

## 二、随机事件系统

### 事件链与因果

事件不是孤立的，而是可以形成链条：
1. 探索 → 发现灵草 → 选择"采摘（得灵石）/ 移植（持续灵气buff）"
2. 受妖兽袭击 → 下次探索可触发"追踪妖兽老巢"（高回报高风险）
3. **限时事件**：秘境开启持续3次行动，不进入则消失

### 沉浸式文案

使用 Rich markup 颜色标记：
- 🟢 正面事件（发现灵草、前辈传功）
- 🔴 负面事件（妖兽袭击、走火入魔）
- 🟡 奇遇事件（秘境、天降机缘）
- 🔵 普通事件（修行感悟）

示例文案：
> 🟡 你在雾隐山谷中发现一株泛着微光的月华草，采集时惊动守护的雾蝎。你艰难击杀后，从蝎尾取得毒囊，在坊市换得 10 灵石。

### 境界分层事件池

- 炼气期：山野精怪、散修切磋、低级灵草
- 筑基期：小门派任务、中级妖兽、坊市拍卖
- 金丹期：探索秘境、斗法大会、天劫预兆

## 三、境界突破系统

### 突破流程

1. **修为满 → 尝试突破**（点击突破按钮或自动触发）
2. 显示："体内灵气澎湃，尝试冲击 [下一层] 瓶颈..."
3. **可选辅助**：服用"护脉丹"（降低失败惩罚，消耗灵石）
4. 突破判定：基础成功率 + 丹药加成 vs 失败反噬
5. 成功 → 境界提升，属性成长，寿元增加
6. 失败 → 修为回退 + 气血伤害（有护脉丹则减轻）

### 大境界突破（天劫）

突破炼气→筑基、筑基→金丹时触发**天劫事件**：
- 三道雷劫，每道需在几秒内选择应对方式
- 三个按钮：「运功硬抗」/「身法躲避」/「法器抵挡」
- 不同选择消耗不同资源（气血/灵气/灵石），成功概率各异
- **增加紧张感和参与感**，不是无脑点突破

### 交互优化

- 所有操作绑定**键盘快捷键**（C/E/R/D/M/S/B），按钮旁显示提示
- 底部消息栏使用 Textual `notify` 机制，弹窗淡出不干扰主区域
- 修为进度条使用 Rich `BarColumn`
- 气血/灵气用双色进度条（当前值/最大值）
- 寿元用 ⏳ 沙漏符号 + 倒计时文本
- 日志面板支持颜色分类，可自动滚动

## 五、存档 & 持久吸引力

### 存档设计

- JSON 格式，每次操作后自动保存到 `data/saves/autosave.json`
- 保留最近 **3 个滚动备份**，防止坏档
- 手动存档/读档（操作菜单按钮）

### 修仙年表

在角色面板中保存修炼历程：
> 第 5 年 → 突破炼气三层
> 第 12 年 → 遭遇妖兽袭击，负伤
> 第 23 年 → 突破筑基期！

### 转世系统（MVP 可省略，但设计预留）

寿元/战死后的多周目机制：
- 新角色继承部分灵石和低级丹药
- 随机继承一项天赋（修炼效率略高）

## 七、NPC 系统架构（Post-MVP 扩展，非 MVP 内容）

> ⚠️ NPC 是 **MVP 之后**的扩展模块。MVP 中世界事件通过 `game/events.py` 的文案渲染，不引入独立 NPC 实体。
> 但代码架构在设计时预留 NPC 接口，确保后续平滑接入。

### 为什么不推荐强化学习

| 问题 | 说明 |
|---|---|
| 状态空间膨胀 | NPC 要跟踪自身属性、玩家关系、世界事件，维度太大 |
| 奖励难设计 | 修仙目标模糊（变强？飞升？长寿？），RL 易学到钻空子行为 |
| 黑箱不可控 | RL 策略可能做出"理性但反直觉"行为，破坏叙事感 |
| 本地开销 | 每次 NPC 行动做一次推理，拖累 TUI 响应 |

### 推荐方案：效用系统 + 有限状态机 + 好感度规则

#### 1. 需求驱动效用 AI

每个 NPC 有 `Needs`（需求）和 `Weights`（偏好系数）：

| 需求 | 触发条件 | 倾向行为 |
|---|---|---|
| 生存 | 气血低 | 休息/吃药 |
| 修炼 | 修为未满 | 闭关 |
| 资源 | 灵石少 | 交易/探索 |
| 社交 | 好感度低 | 论道/送礼 |
| 探索 | 好奇心高 | 去危险区域 |

每回合计算 `score = 需求紧迫度 × 偏好系数 + 随机抖动(±15%)`，选最高分执行。

#### 2. 有限状态机

```
日常修炼 ↔ 敌对状态（攻击过玩家）
    ↓            ↓
濒死状态 ──→ 逃跑或求饶
    ↑
机缘状态（探索到秘境，暂离）
```

状态切换由**世界事件**或**玩家交互**触发。

#### 3. 好感度规则系统

- 送丹药、论道、组队 → 好感↑
- 抢夺资源、伤其亲友 → 好感↓
- 阈值行为：>80 传功/双修，<0 拒绝交易，<-50 联合围攻

#### 4. 架构预留

- 新建 `game/npc.py`：`NPCState` 类（类似 `PlayerState`）+ 效用计算
- `game/actions.py` 增加玩家-NPC 交互：论道、挑战、赠礼
- `game/events.py` 增加 NPC 驱动事件：NPC 突破公告、NPC 死亡遗产
- 数量控制：10~20 个关键 NPC，时间推进时遍历执行
- UI 日志用颜色展示 NPC 动态：`[金] 太虚散人在青冥山渡劫成功，踏入金丹期！`

## 八、词条组合生成系统设计（Post-MVP 扩展）

> 用随机词条组合生成 NPC/物品/地图名称，标签（Tag）系统赋予属性。
> 纯文本处理，零额外硬件负担，让每次新档都有独特的探索体验。

### 1. 词条库结构（`game/config.py` 或独立 `game/lexicon.py`）

| 词库 | 示例 | 用途 |
|---|---|---|
| **前缀** | 焚天、幽冥、万古、裂空、玉液、枯荣、玄黄 | 描述特性 |
| **核心** | 剑、丹、经、泉、林、蛇、鼎、脉、骨 | 定义类别 |
| **后缀** | 真解、秘录、宝地、散人、上人、谷、渊、峰 | 补充意境 |

**组合规则**：
- 地图：`[前缀] + [核心] + [后缀]` → 「焚天炎谷」「幽冥骨渊」
- 物品：`[前缀] + [核心]丹/诀` → 「玉液化厄丹」「混元炼心诀」
- NPC：`[核心] + [后缀]` → 「玄骨上人」「青叶散人」

### 2. 标签系统（关键词 → 属性映射）

| 关键词 | 标签 | 效果示例 |
|---|---|---|
| 火/焚/炎/熔 | `fire` | 灼烧伤害，克木 |
| 冰/寒/霜/玄 | `ice` | 减速，冰系加成 |
| 毒/瘴/腐 | `poison` | 持续伤害，丹药带毒 |
| 古/荒/太初 | `ancient` | 品质更高，附带未知效果 |
| 血/骨/怨 | `cursed` | 强力但有负面代价 |
| 灵/圣/仙 | `divine` | 纯正面，稀有 |
| 金/木/水/火/土 | `element:N` | 五行属性克制 |

**实现**：生成名字时扫描关键词 → 自动打标签 → 标签组合动态计算属性。

### 3. 地图生成

- 新游戏用 **种子（seed）** 生成区域节点图
- 布局：5~8 个区域，每个区域由词条组合命名
- 区域标签决定：灵气浓度、资源类型、怪物种类、事件概率
- 用节点列表/连线表示连通关系（非网格）
- 玩家探索时事件根据区域标签调整概率

### 4. 物品/NPC 随机化

- **NPC**：名字随机 → 标签决定初始性格（带"煞"的攻击性高）、修炼功法、携带物品
- **丹药**：类型决定基础效果（回血/回灵/加修为），标签添加附加属性
- **法器**：基础属性 + 标签词缀（灵纹=灵气回复，血纹=吸血）
- **稀有度**：由正面标签数量决定，负面标签可随机出现制造"有代价的强力道具"

### 5. 叙事嵌入

> 你在「焚天炎谷」的裂缝中发现一株「炎髓芝」，采集时遭到「赤鳞火蟒」袭击。
> 击败后，从其守护的「玉液化毒丹」炼制手札中习得新配方。

全部名称由系统实时组合，玩家感知到一个鲜活丰富的世界。

### 6. 注意事项

- **避免无意义随机**：基础语法约束 + 可读性审查，放弃不和谐组合
- **标签不膨胀**：初期 8~12 个核心标签（五行 + 圣/邪/古/毒）
- **持久化**：首次生成后保存 seed 或结果，保证存档稳定
- **玩家命名权**：允许玩家自己命名发现的物品，系统补标签

## 九、MVP vs Post-MVP 路线图

### MVP 范围（本次实现）

| 模块 | 状态 |
|---|---|
| 仪表盘 UI（Textual 四面板） | ✅ |
| 核心修炼循环（修炼/探索/休息/炼丹/坊市） | ✅ |
| 资源拮抗（气血/灵气/寿元/灵石） | ✅ |
| 随机事件（带颜色 + 因果链） | ✅ |
| 境界突破 + 天劫雷击 | ✅ |
| 存档/读档 + 年表 | ✅ |
| 批量闭关 | ✅ |
| 游戏结束 + 年表回顾 | ✅ |

### Post-MVP 扩展（预留接口）

| 模块 | 优先级 |
|---|---|
| 词条组合生成系统（名称/物品/地图/NPC） | ⭐⭐⭐ |
| NPC 实体系统（效用AI+状态机+好感度） | ⭐⭐⭐ |
| 功法系统（影响修炼效率/战斗） | ⭐⭐ |
| 炼器/制符（装备制造） | ⭐⭐ |
| 转世/多周目（继承部分资源） | ⭐⭐ |
| 宗门/门派系统 | ⭐ |

### 运行节奏

- **批量修炼模式**：防后期重复劳动（MVP 包含）
- **快捷键**：所有操作绑定键盘快捷键（MVP 包含）

### 批量修炼模式（防后期重复劳动）

操作菜单中「批量闭关」按钮 → 连续执行修炼 N 次（5/10/20次可选）：
- 跳过中间日志，只显示总结（修为总增长 + 关键事件）
- 若期间触发限时事件则中断批量

### 模块化扩展接口

`game/actions.py` 和 `game/events.py` 设计为纯函数式数据流：
- 输入 `(state, action_params)` → 输出 `(new_state, messages[])`
- 未来扩展方向（不影响现有结构）：

| 模块 | 影响 |
|---|---|
| 功法系统 | 修炼效率、战斗能力 |
| 炼器/制符 | 装备制造，属性加成 |
| NPC/宗门 | 长期 buff、任务系统 |
| 灵根天赋 | 开局差异化 |

## 七、MVP 实现步骤

### Step 1: 项目骨架
- `pyproject.toml`（textual>=8.0）
- 目录结构搭建

### Step 2: 游戏数据层 `game/`
- `config.py` — 境界体系、事件表、丹药效果、数值公式
- `state.py` — `PlayerState` dataclass + 序列化
- `actions.py` — 修炼/探索/休息/炼丹/坊市/批量闭关
- `events.py` — 事件链引擎、境界分层事件池、限时事件
- `save.py` — JSON 存档/读档/滚动备份 + 修仙年表记录

### Step 3: UI 层 `ui/`
- `app.py` — Textual App 主类
- `screens.py` — 标题画面、主游戏画面、突破动画、游戏结束
- `widgets.py` — StatusPanel / LogPanel / ActionMenu / MessageBar

### Step 4: 突破系统
- 小突破（层内）：概率 + 丹药辅助
- 大突破（天劫事件）：雷劫应对选择

### Step 5: 打磨
- 快捷键绑定
- 通知动画
- 批量闭关优化
- 修仙年表查看
- 转世系统（可选）
